## 🎯 CORE FUNCTIONALITY

### **What This App Does:**
1. **Admin creates rooms** (e.g., "Room 201", "Room 202")
2. **Each room gets a unique QR code** displayed on a TV/monitor
3. **Users scan QR code with phone** → opens upload page
4. **Users upload images from phone** → images appear on TV display
5. **Images auto-rotate** every 10 seconds if multiple images uploaded
6. **Admin can also upload** images directly from admin panel (no QR needed)
7. **Clear button** removes all images and returns to QR code
8. **Auto-purge** deletes old images after 48 hours (configurable)

### **1. app.py** (Main Flask Application - Backend)

**Purpose:** Core backend server handling all routes, API endpoints, database operations, and file management.

#### **Imports & Configuration:**
```python
from flask import Flask, render_template, request, jsonify, send_from_directory
from flask_cors import CORS
import sqlite3
import os
import uuid
from datetime import datetime, timedelta
from apscheduler.schedulers.background import BackgroundScheduler
```

#### **Configuration Variables:**
```python
UPLOAD_FOLDER = 'uploads'
DATABASE = 'qr_rooms.db'
SERVER_IP = os.getenv('SERVER_IP', '10.192.128.125')
SERVER_PORT = os.getenv('SERVER_PORT', '5000')
SERVER_URL = f'http://{SERVER_IP}:{SERVER_PORT}'
AUTO_PURGE_ENABLED = os.getenv('AUTO_PURGE_ENABLED', 'true').lower() == 'true'
AUTO_PURGE_HOURS = int(os.getenv('AUTO_PURGE_HOURS', '48'))
```

#### **Database Schema:**

**Table: `rooms`**
```sql
CREATE TABLE rooms (
    id TEXT PRIMARY KEY,              -- 8-character UUID (e.g., "a3f2b9c1")
    name TEXT NOT NULL,               -- Room name (e.g., "Room 201")
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

**Table: `room_images`**
```sql
CREATE TABLE room_images (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    room_id TEXT NOT NULL,            -- Foreign key to rooms.id
    image_path TEXT NOT NULL,         -- File path (e.g., "uploads/a3f2b9c1_abc123_photo.jpg")
    uploaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (room_id) REFERENCES rooms (id) ON DELETE CASCADE
)
```

#### **Functions:**

**`get_db()`**
- Opens SQLite database connection
- Sets row_factory to sqlite3.Row for dict-like access
- Returns database connection object

**`init_db()`**
- Creates database tables if they don't exist
- Called on app startup
- Ensures schema is ready

**`purge_old_images()`**
- Runs automatically every hour (if AUTO_PURGE_ENABLED=true)
- Deletes images older than AUTO_PURGE_HOURS (default: 48 hours)
- Removes files from filesystem AND database entries
- Logs purge activity

**`start_scheduler()`**
- Initializes APScheduler background scheduler
- Schedules `purge_old_images()` to run every hour
- Only runs if AUTO_PURGE_ENABLED=true

#### **Routes (Web Pages):**

**`GET /`**
- Renders admin panel (`admin.html`)
- Passes `server_url` to template

**`GET /admin`**
- Alternative route to admin panel
- Same as `/`

**`GET /display/<room_id>`**
- Renders display page for specific room (`display.html`)
- Shows QR code or uploaded images
- Passes `room_id` and `server_url` to template

**`GET /upload/<room_id>`**
- Renders mobile upload page (`upload.html`)
- Accessed by scanning QR code
- Passes `room_id` and `server_url` to template

#### **API Endpoints:**

**`GET /api/rooms`**
- Returns JSON array of all rooms
- Ordered by created_at DESC (newest first)
- Response: `[{id, name, created_at}, ...]`

**`POST /api/rooms`**
- Creates new room
- Request body: `{name: "Room 201"}`
- Generates 8-character UUID for room_id
- Response: `{success: true, room_id: "a3f2b9c1", name: "Room 201"}`

**`DELETE /api/rooms/<room_id>`**
- Deletes room and all associated images
- Removes image files from filesystem
- Deletes database entries (cascade)
- Response: `{success: true}`

**`POST /api/save_images/<room_id>`**
- Uploads images for a room
- Accepts multiple files via `images[]` form field
- Clears existing images before saving new ones
- Saves files to `uploads/` directory
- Filename format: `{room_id}_{uuid}_{original_filename}`
- Response: `{success: true, count: 3}`

**`GET /api/get_images/<room_id>`**
- Returns array of image URLs for a room
- Ordered by uploaded_at ASC (oldest first for slideshow)
- Response: `{images: ["http://10.192.128.125:5000/uploads/image1.jpg", ...]}`

**`POST /api/clear_images/<room_id>`**
- Clears all images for a room
- Deletes files from filesystem
- Deletes database entries
- Response: `{success: true}`

**`GET /uploads/<filename>`**
- Serves uploaded image files
- Static file serving from uploads directory

#### **App Initialization:**
```python
if __name__ == '__main__':

    if AUTO_PURGE_ENABLED:
        start_scheduler()        # Start auto-purge scheduler
```

### **3. Dockerfile** (Docker Container Configuration)

**Purpose:** Defines how to build the Docker container image.

```dockerfile
FROM python:3.11-slim           # Base image: Python 3.11 (lightweight)

WORKDIR /app                    # Set working directory inside container

COPY requirements.txt .         # Copy dependencies file
RUN pip install --no-cache-dir -r requirements.txt  # Install Python packages

COPY . .                        # Copy all application files

RUN mkdir -p uploads            # Create uploads directory

EXPOSE 5000                     # Expose port 5000 to host

CMD ["python", "app.py"]        # Run Flask app on container start
```

**What happens:**
1. Starts with Python 3.11 slim image (smaller size)
2. Installs all Python dependencies
3. Copies application code into container
4. Creates uploads directory
5. Exposes port 5000
6. Runs `python app.py` when container starts

### **4. docker-compose.yml** (Docker Compose Orchestration)

**Purpose:** Defines how to run the Docker container with proper configuration.

```yaml
version: '3.8'

services:
  qr-image-transfer:
    build: .                                    # Build from Dockerfile in current directory
    container_name: qr-image-transfer          # Container name
    ports:
      - "5000:5000"                            # Map host port 5000 to container port 5000
    volumes:
      - ./uploads:/app/uploads                 # Mount uploads directory (persist images)
    env_file:
      - .env                                   # Load environment variables from .env
    restart: unless-stopped                    # Auto-restart on failure (not on manual stop)
```

**Key Points:**
- Database is INSIDE container (not mounted) - fixes permission errors
- Only uploads directory is mounted - persists images across container restarts
- Reads .env file for configuration
- Auto-restarts if crashes

### **17. static/js/display.js** (Display Page Logic)

**Purpose:** JavaScript for TV display page (QR code and images).

#### **Global Variables:**
```javascript
let currentImageIndex = 0;          // Current image in slideshow
let images = [];                    // Array of image URLs

```

#### **Functions:**

**`generateQRCode()`**
- Creates QR code using QRCode.js library
- QR code contains upload URL: `http://10.192.128.125:5000/upload/{roomId}`
- Size: **300x300 pixels** (v6.1 fix)
- Error correction: High (QRCode.CorrectLevel.H)
- Colors: Black on white

```javascript
function generateQRCode() {
    const qrDiv = document.getElementById('qrcode');
    const uploadUrl = `${serverUrl}/upload/${roomId}`;
    qrDiv.innerHTML = '';

    new QRCode(qrDiv, {
        text: uploadUrl,
        width: 300,
        height: 300,
        colorDark: '#000000',
        colorLight: '#ffffff',
        correctLevel: QRCode.CorrectLevel.H
    });
}
```

**`checkForImages()`**
- Fetches images from API (`GET /api/get_images/{roomId}`)
- If images exist: calls `showImages()`
- If no images: calls `showQRCode()`
- Called every 3 seconds via `setInterval`

**`showQRCode()`**
- Hides image container
- Shows QR code container
- Stops slideshow

**`showImages()`**
- Hides QR code container
- Shows image container
- Resets to first image
- Starts slideshow if multiple images

**`displayCurrentImage()`**
- Updates `<img>` src to current image URL
- Uses `currentImageIndex` to select image

**`startSlideshow()`**
- Stops existing slideshow (if any)
- Creates interval timer (10 seconds)
- Increments `currentImageIndex` (wraps around)
- Calls `displayCurrentImage()`

**`stopSlideshow()`**
- Clears interval timer
- Sets `slideshowInterval` to null

**`clearImages()`**
- Shows confirmation dialog
- Sends POST to `/api/clear_images/{roomId}`
- Clears local images array
- Stops slideshow
- Shows QR code

- Creates `<img>` elements using FileReader
- Displays in preview grid
- Shows upload button

**`uploadImages()`**
- Validates at least one file selected
- Creates FormData object
- Appends all files with key `images[]`
- Sends POST to `/api/save_images/{roomId}`
- Disables button during upload
- Shows success/error status
- Auto-closes window after 2 seconds on success

**`showStatus(message, type)`**
- Displays status message
- Types: 'success' (green) or 'error' (red)
- Updates `#status` div

#### **Event Listener:**
```javascript
document.getElementById('fileInput').addEventListener('change', function(e) {
    selectedFiles = Array.from(e.target.files);
    displayPreview();

    if (selectedFiles.length > 0) {
        document.getElementById('uploadBtn').style.display = 'block';
    }
});
```

### **Workflow 2: User Uploads via QR Code (Mobile)**

2. Phone opens `http://10.192.128.125:5000/upload/{roomId}`
3. User taps upload area
4. Selects one or more images from phone
5. Previews appear
6. User taps "Upload Images"
7. **Backend:** Saves files to `uploads/`, inserts into `room_images` table
8. **Frontend:** Shows success message, closes after 2 seconds
9. **Display page:** Polls API, detects new images, switches from QR to images
10. If multiple images: auto-rotates every 10 seconds

### **Workflow 3: Admin Uploads Directly**

1. Admin opens `http://10.192.128.125:5000`
2. Clicks "Upload Images" button on room card
3. Modal pops up
4. Selects images from computer
5. Previews appear in modal
6. Clicks "Upload Images"
7. **Backend:** Same as Workflow 2
8. **Display page:** Same as Workflow 2

---

### **Workflow 4: Clear Images**

1. User clicks "Clear" button on display page
2. Confirmation dialog appears
3. User confirms
4. **Backend:** Deletes files from `uploads/`, deletes from `room_images` table
5. **Display page:** Returns to QR code

---

### **Workflow 5: Auto-Purge (Background)**

1. Scheduler runs `purge_old_images()` every hour
2. Queries `room_images` for entries older than 48 hours
3. Deletes files from filesystem
4. Deletes database entries
5. Logs purge activity
