## 📋 EXECUTIVE SUMMARY

**Version:** 7.0.0  
**Type:** Flask Web Application (Python)  
**Deployment:** Docker Container  
**Default Server IP:** 192.168.1.197 (configurable)  
**Port:** 5000 (configurable)  
**Database:** SQLite (persistent at ./data/qr_rooms.db)  
**Purpose:** Multi-room QR code system for uploading and displaying images on TV/monitor displays with admin search/sort

---

## 🎯 CORE FUNCTIONALITY

### **What This App Does:**
1. **Admin creates rooms** (e.g., "Room 201", "Room 202") with search and sort controls
2. **Each room gets a unique QR code** displayed on a TV/monitor
3. **Users scan QR code with phone** → opens upload page
4. **Users upload images from phone** → images appear on TV display
5. **Images auto-rotate** every X seconds (configurable via SLIDE_DURATION env var)
6. **Admin can also upload** images directly from admin panel (no QR needed)
7. **Clear button** removes all images; admin version requires confirmation
8. **Search and filter** rooms by name or ID with live results
9. **Sort rooms** by creation date (newest/oldest) or name (A→Z/Z→A)
10. **Auto-purge** deletes old images after 48 hours (configurable)
11. **Database persists** across container rebuilds via mounted ./data volume

## 🔄 USER WORKFLOWS (v7.0.0)

---

### **Workflow 1: Admin Creates and Searches Rooms**

1. Admin opens `http://192.168.1.197:5000/admin`
2. Enters room name (e.g., "Room 201")
3. Clicks "Create Room"
4. **Backend:** Generates 8-char UUID, inserts into `rooms` table
5. **Frontend:** Room card appears in grid with search results updated
6. Admin types in search box to filter rooms (live, debounced 250ms)
7. Admin clicks sort button to change order (Newest/Oldest/Name A→Z/Name Z→A)
8. Search term is highlighted in room name and ID
9. Results count shows matching rooms

---

### **Workflow 2: Admin Clears Images from Room (v7.0.0)**

1. Admin clicks "Clear" button on room card
2. Confirmation dialog: "Clear all images from this room?"
3. Admin confirms
4. **Backend:** Deletes files from `uploads/`, deletes from `room_images` table
5. **Frontend:** Room card reloads (if display was showing images, returns to QR)

---

### **Workflow 3: User Uploads via QR Code (Mobile)**

1. User scans QR code with phone camera
2. Opens `http://192.168.1.197:5000/upload/{roomId}`
3. User taps upload area
4. Selects one or more images
5. Previews appear
6. User taps "Upload Images"
7. **Backend:** Saves files, inserts into `room_images` table
8. **Frontend:** Shows success message
9. **Display page:** Polls API, detects images, switches from QR to images
10. If multiple images: auto-rotates every 5 seconds (configurable via SLIDE_DURATION)

---

### **Workflow 4: Display Updates Images**

1. Display page shows QR code
2. User uploads images via phone
3. Display page polls API every 3 seconds
4. Detects new images
5. Switches to image display
6. Starts slideshow with configured duration
7. When Clear is clicked on display: removes images, returns to QR
   - v7.0.0: No confirmation dialog (tablet-friendly)
8. When Clear is clicked on admin: confirmation required before removal

---

### **Workflow 5: Auto-Purge Background Task**

1. Scheduler runs `purge_old_images()` every hour
2. Queries `room_images` for entries older than 48 hours (configurable)
3. Deletes files from filesystem
4. Deletes database entries
5. Logs activity

## 🔒 WHAT'S SECURE vs. NOT SECURE

### **Current Security (v7.0.0):**
- ❌ No admin authentication (anyone can access admin panel)
- ❌ No HTTPS (HTTP only - fine for local network)
- ❌ No rate limiting (no protection against DoS)
- ❌ No file type validation (depends on browser)
- ❌ No CSRF protection
- ✅ CORS enabled for API
- ✅ 50MB file size limit
- ✅ UUID room IDs (not sequentially guessable)
- ✅ Database inside container (isolated)
- ✅ Images not world-readable (mounted volume only)

### **Production Recommendations:**
- Add HTTP Basic Auth for admin panel
- Use HTTPS with self-signed or CA certificate
- Add Flask-Limiter for rate limiting
- Validate file MIME types server-side
- Implement CSRF protection tokens
- Add user session management
- Use firewall rules to restrict access

## 📈 PERFORMANCE CHARACTERISTICS

- **Admin page load:** <100ms (local query, client-side sort/filter)
- **Display QR load:** ~50ms (just generates QR code)
- **Image upload:** Depends on file size; 50MB limit per file
- **Slideshow polling:** 3-second intervals (API call + DOM update)
- **Auto-purge:** 1-hour intervals (runs async in background)
- **Search debounce:** 250ms (prevents excessive re-renders)
- **Database:** SQLite (single-file, suitable for <10K rooms)

### **Scalability Limits:**
- **Rooms:** Unlimited (but admin page may slow with 10K+ rooms)
- **Images per room:** Unlimited (but storage disk space is limit)
- **Image size:** 50MB max per file
- **Concurrent users:** ~50 (depends on network bandwidth)
- **Recommended:** <2000 rooms, <500 images per room, <100GB storage

## 📝 API REFERENCE (v7.0.0)

All endpoints respond with JSON.

### **GET /api/rooms**
Returns array of all rooms ordered by creation date (newest first).

Response:
```json
[
    {
        "id": "a3f2b9c1",
        "name": "Room 201",
        "created_at": "2026-02-19 10:30:45"
    },
    {
        "id": "b4f3c9d2",
        "name": "Room 301",
        "created_at": "2026-02-19 09:15:22"
    }
]
```

### **POST /api/rooms**
Create new room.

Request:
```json
{
    "name": "Room 201"
}
```

Response:
```json
{
    "success": true,
    "room_id": "a3f2b9c1",
    "name": "Room 201"
}
```

### **DELETE /api/rooms/{room_id}**
Delete room and all associated images.

Response:
```json
{
    "success": true
}
```

### **POST /api/save_images/{room_id}**
Upload images for a room (replaces existing images).

Request: FormData with `images[]` files

Response:
```json
{
    "success": true,
    "count": 3
}
```

### **GET /api/get_images/{room_id}**
Get all image URLs for a room (ordered by upload time).

Response:
```json
{
    "images": [
        "http://192.168.1.197:5000/uploads/a3f2b9c1_abc123_photo1.jpg",
        "http://192.168.1.197:5000/uploads/a3f2b9c1_def456_photo2.jpg"
    ]
}
```

### **POST /api/clear_images/{room_id}**
Remove all images for a room.

Response:
```json
{
    "success": true
}
```

## 🔧 TROUBLESHOOTING

### **Database won't create**
- Old issue (v6.2): Ensure `./data` folder exists on host: `mkdir data`
- Check permissions: `ls -la data`
- Container `init_db()` should create `data/qr_rooms.db`

### **Search not working**
- Refresh browser page
- Check browser console for JavaScript errors
- Search is client-side and case-insensitive

### **Images not sliding**
- Check `SLIDE_DURATION` in .env (milliseconds)
- Ensure multiple images are uploaded
- Check browser console for errors

### **Clear button doesn't respond**
- On display page: No confirmation, should clear immediately
- On admin page: Should ask "Clear all images from this room?"
- Refresh page if stuck

### **Port conflict**
- Change `SERVER_PORT` in .env
- Change `"5000:5000"` in docker-compose.yml to `"8080:5000"` (or desired port)
- Restart: `docker compose up -d`

### **Can't access from other computers**
- Ensure firewall allows port 5000
- Use actual server IP (not localhost)
- Check `SERVER_IP` in .env matches your network
